Accesibilidad Web
  • Principios
  • Referencias

1-Perceptible

La interfaz de la página web se debe de presentar de forma que cualquier usuario pueda percibir todo el contenido. El diseño del sitio debe de presentarse de forma que los diferentes elementos de la página puedan ser distinguidos entre sí.

Ejemplos

• Utilizar adaptaciones de contenido como zoom u orientaciones de pantalla.

• Uso de subtítulos y audios para adaptar la página para aquellos usuarios con limitaciones visuales o auditivas.

• Asegurar la distinción entre los componentes o elementos de la página, por ejemplo, utilizando contrastes y colores adecuados .

Scope aplica frases cortas y sencillas, combinadas con un adecuado uso de colores y fuentes.

2-Operable

Este principio hace referencia a que un usuario pueda interactuar plenamente con el sitio, es decir, no debe encontrar alguna limitación al navegar o realizar una acción en la página.

Ejemplos

• Para usuarios sin ratón o alguna otra limitación, poder contar con la opción del teclado para garantizar el uso de todas las funcionalidades.

• Facilitar al usuario el ingreso de datos, mediante el uso de otros métodos de entrada adicionales al teclado.

• El usuario debe contar con el tiempo suficiente para leer e interactuar con el contenido del sitio.

Paralympic.org cuenta con un panel en la esquina superior derecha del sitio, con opciones para ajustar la operabilidad y percepción de la página.

3-Comprensible

El usario debe encontrar en el sitio web un espacio que proporcione la posiblidad de interactuar con una interfaz con un contenido entendible, es decir, la información debe ser fácil de leer y de comprender.

Ejemplos

• Diseñar el sitio web con un funcionamiento predecible.

• Ofrecer al usuario ayuda o una guía al ingresar datos.

• Mostrar al usuario ejemplos de uso o formato, en acciones concretas como al completar un formulario.

• Utilizar un texto legible y fácil de comprender.

Harward no solo se destaca por su calidad en carreras y educación, sino también por la alta accesibilidad en su sitio web. El uso de textos comprensibles, y fuentes de tamaño adecuado, combinado con el uso de herramientas para adaptar el texto, brindan al usuario una mejor experiencia y facilitan la comprensión de la información.

4-Robusto

Se debe procurar que el sitio web cuente con un contenido robusto y bien organizado, es decir, el contenido debe ser seguro y confiable para el usuario. Este principio pretende que asegurar que el contenido de la página sea lo suficientemente accesible y utilizable para cualquier usuario, independientemente de los dispositivos o tecnologías que los usuarios utilicen. Esto garantiza que ante los avances tecnológicos los usuarios con discapacidades no se queden atrás.

Ejemplos

• Uso de software de apoyo como lectores de pantalla.

• Uso de atributos ALT en imágenes, para que el usuario cuente con descripciones significativas del contenido.

• Uso correcto de encabezados (h1,h2,h3) y utilizar un CSS responsivo.

Bleacher Report ofrece al usuario la posibilidad de utilizar lectores de pantalla o controles de teclado.
